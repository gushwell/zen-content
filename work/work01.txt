## ✅ このページに書かれている重要ポイント

### **1. ユーザー入力データ（プロンプト・出力）の取り扱い**

ページでは次のように明確に書かれています：

*   **あなたのプロンプトや出力は他の顧客には提供されません。**
*   **OpenAIや他のモデル提供者にも利用されません。**
*   **あなたの許可なしに基盤モデルの訓練には使用されません。**
*   **Microsoftや第三者のサービス改善のためにも利用されません。**

つまり、  
➡ **入力データが勝手に学習に使われることはない**  
➡ **他の企業・外部に送られることもない**

と明言されています。

（出典：取得したページ内容中の “あなたのプロンプト(入力)と完了(出力)…は他の顧客には提供されていません…あなたの許可や指示なしに生成AI基盤モデルの訓練には使用されません。”）

***

## ✅ 2. データが外部に漏れる可能性について

ページの記載では：

*   データは **Azure のサービス境界の中で処理される**
*   プロンプトやレスポンスは **指定した地理（リージョン）内で処理**
*   **モデルはユーザーデータを保存しない（ステートレス）**
*   **人間が閲覧するのは「虐待・不正検知でフラグが立った場合のみ」**

つまり通常は  
➡ **ユーザーの入力データが外部に漏れる設計ではない**  
と記述されています。

ただし例外として、不正利用の疑いがあると判断された場合のみ、  
Microsoft の認可された担当者が手動レビューを行う可能性があります  
（これもページに明記あり）。

***

## ✅ 3. データが保存されるケース

保存されるのは **以下の「ユーザーが明示的に使う機能」だけ**：

*   ファインチューニングのためにアップロードしたデータ
*   ベクターストア
*   Files API のアップロードデータ
*   スレッド履歴（Assistants API / Responses API）
*   Stored completions（保存完了）

これらはすべて  
➡ **ユーザー自身の Azure テナント内で保存され、外部に渡らない**  
と記載されています。
